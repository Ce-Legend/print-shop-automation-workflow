# Workflow Design

这套工作流从打印店现场的动作开始设计：订单文件夹进入监控目录后，系统解析命名、检查图片、选择预设、进入队列、执行打印，并把异常留在可复核的位置。

## 文件夹名就是任务单

订单信息不再额外维护表格，而是写在文件夹名里。这样店员只需要移动文件夹，系统负责解析：

```text
【拍立得】5寸,10张 250701-123456789012345
【全景】6寸,8张 250702-123456789012346
3T_拍立得_5张250703-123456789012347
```

解析结果包含尺寸、模式、张数和订单号。尺寸决定纸张和打印预设，模式决定是否进入拍立得预处理，订单号用于日志、归档和异常追踪。

## 预设映射

配置里用尺寸和模式映射打印预设：

```json
{
  "5寸": {
    "全景": "5寸全景",
    "拍立得": "5寸拍立得",
    "default": "5寸拍立得"
  }
}
```

模式缺失时使用 `default`。这能覆盖一线常见情况：文件夹名里写了尺寸和订单号，但没有明确标注拍立得或全景。

## 异常恢复

异常不直接丢掉，也不静默跳过。系统把问题分成几类：

- 缺少尺寸：无法匹配纸张。
- 缺少订单号：无法追踪归档。
- 没有图片：无法进入队列。
- 打印机不可用：等待人工检查设备或预设。

每个异常都应该能被店员看到，修正文件夹名或设备状态后再重新放入监控目录。

## 语音和界面提示

门店现场没人一直看控制台。系统同时使用 UI 状态、日志和语音提示，让操作员知道任务进入队列、完成、失败或需要人工处理。

语音只做状态提醒，不承载复杂解释。复杂原因留在日志和异常文件夹里。
