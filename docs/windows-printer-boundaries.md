# Windows Printer Boundaries

这个项目依赖 Windows 本地打印能力。它能自动化一部分重复工作，也需要承认打印机驱动的边界。

## 程序能稳定接管的部分

- 监控订单文件夹。
- 解析尺寸、模式、张数和订单号。
- 对图片做格式检查和拍立得预处理。
- 根据配置选择打印机和预设名称。
- 维护任务队列、日志、异常状态和完成归档。

这些部分适合写进代码和测试，因为它们不依赖某一台具体打印机。

## 需要人工配置的部分

很多照片打印机的色彩、边距、纸张、驱动高级选项藏在厂商 UI 里。Windows 标准接口不能稳定覆盖所有驱动私有参数。

所以项目采用一次配置、持续复用的方式：

- 先在本机驱动里配置好纸张和色彩预设。
- 在 `config.json` 里把尺寸和模式映射到预设名称。
- 系统按订单选择预设，遇到预设缺失时进入异常流程。

## 验证建议

上线前至少做三类本地验证：

- 不接打印机：跑解析和配置测试，确认任务会进入正确队列。
- 接测试打印机：用 1-2 张图片验证纸张、边距和方向。
- 接真实打印机：用门店常见订单名跑一轮，检查日志、语音提示和异常恢复。

公开仓库保留的是通用工作流。真实打印机名称、驱动预设和门店路径应放在本机 `config.json`，不要提交到仓库。
